# Initial Concept

The project aims to build a comprehensive, multi-tool guide for AI-powered development agents. It provides a structured, deep-dive into tools like Cursor, Antigravity, Gemini CLI, OpenCode, Claude Code, and Codex CLI, focusing on practical implementation, best practices, and model evaluations.

# Product Guide: AI-Agents-Guide

## 1. Vision & Strategy
A "one-stop-shop" reference for modern AI agent development. It moves beyond simple documentation by providing:
- **Consistent Structure:** Every tool (Cursor, Claude Code, etc.) is documented with identical sections (What is it, How to use, Best practices, etc.).
- **Comparative Insights:** Direct comparisons between models (Gemini, Claude, GPT) and tools to help users choose the right solution for their specific tasks.
- **Ready-to-Use Templates:** Practical examples for Skills and Workflows that users can copy and adapt.

## 2. Target Audience
This guide is designed for a broad spectrum of users:
- **Beginners to AI Agents:** Those looking for clear definitions and step-by-step setup instructions.
- **Advanced AI Engineers:** Developers seeking deep dives into MCP architectures, hooks, and complex subagent workflows.
- **Full-Stack Developers:** Engineers looking for practical snippets, tool comparisons, and "golden rules" to integrate AI into their daily development.

## 3. Key Features & Priority Sections
The guide will prioritize the following areas:
- **Skills Deep Dive:** Comprehensive documentation on creating, managing, and optimizing agent skills, including a dedicated template library.
- **MCP & Tools Ecosystem:** Detailed coverage of the Model Context Protocol, focusing on security, tool configuration, and common integrations (GitHub, ClickUp, etc.).
- **Model Evaluation:** Data-driven comparisons of top models (Gemini, Claude, GPT) for coding, documentation, and logic tasks.
- **Agents.md / Context Management:** Rules for persistent memory and context isolation.
- **Sub-Agents & Teams:** Strategies for delegating complex tasks to specialized, autonomous agents.

## 4. Tone and Style
- **Conversational & Educational:** Engaging and approachable, using real-world examples, helpful tips, and architectural warnings to guide the reader.
- **Technically Precise:** While conversational, the guide remains accurate and grounded in verified tool documentation and search results.
- **English-First:** All code comments and technical documentation will be in English for universal compatibility.
